## 1. 定位与边界

### 1.1 一句话定位

**「固定六级词汇底盘 + 职场高频表达补丁」**：每天几分钟，在同一词域内完成「见到词、见到能用的句、会前能念的脚本」，收藏形成可复访的个人语料；**产品内不提供**四级、雅思、考研等多级别切换。

### 1.2 目标人群（核心）

- **夹心层**：人在国内或中英混合环境，英文不是主业，但每周会碰到几次：英文邮件/IM、文档、跨时区会议、对外简单说明。  
- **诉求**：读慢点能懂、写短点能发、开口不断片；**怕丢人** 大于 **怕考不好**——六级刻度作为 **国内职场读写常见难度的统一锚点**，而非强调「备考叙事」。

### 1.3 明确不在首版主线

- 多级别词库、用户可见的级别切换与「推荐升级下一级」。  
- 生活跨境办事（租房/医疗等）作为 **独立主线**（可远期以主题包形式出现）。  
- 重口语陪练、B2B 班级管理、多语言产品线。  
- 「对外人设/简历拼装」可远期或小主题包承接，**不占首版主导航**。

### 1.4 能力契约（与实现对齐，不可回退）

以下能力在演进中 **须保持可用、体验不劣化**：

| 能力 | 用户价值 |
|------|----------|
| **随机抽取** | 低门槛进入学习；随机范围默认来自 **固定六级词域** |
| **单词/卡片流** | 统一呈现面：音、形、义、例句及（分期）句块、脚本、双行补丁 |
| **收藏（红心）** | 「对我重要」→ 写作夹 / 会前夹 / 补丁夹等分夹或标签 |

## 2. 产品支柱（立项校验）

所有功能须能回答：**服务于哪一支柱**。

| 支柱 | 含义 | 本方案下的载体 |
|------|------|----------------|
| **P1 接触（Exposure）** | 持续、低压力地「见到」英语 | 今日短会话；随机默认六级池；可选「职场加浓」子池（仍在六级范围内） |
| **P2 理解（Understanding）** | 在上下文中弄懂含义与用法 | 词卡例句；句块卡；脚本变体；中英双行补丁卡 |
| **P3 个人语料（My English）** | 把重要材料收拢、可复访 | 收藏列表 + 分夹/标签；从收藏生成复习队列（分期） |
| **P4 节奏（Rhythm）** | 习惯与轻反馈：今天完成了什么 | 微完成（默认 3～10 张可配置）；连续「接触英语天数」；轻量统计，无排行榜羞辱 |

## 3. 内容策略：一种词域，多种卡片模板

**词域**：仅 **六级核心/考纲高频子集**（具体词表以内容规格为准）；不向用户暴露其他考试级别。

| 模板类型 | 说明 | 来源合并 |
|----------|------|----------|
| **词条卡** | 六级词 + 职场友好例句（读文档、邮件、会议常见搭配） | 主 PRD + 六级锚定 |
| **句块卡** | 正面可半句/整句（邮件、IM 可直接用），背面拆词 | 《职场人-书面与异步沟通》 |
| **脚本卡** | 主句 + 更正式/更短两变体；场景收敛为站会、对齐、没听清、跟进等 | 《职场人-开口与临场》 |
| **补丁卡** | 上行中文场景或任务，下行最短英文（词或短语） | 《夹心层-中文环境里的英文》 |

**随机策略**：

- 默认：**六级全池弱随机**（保证词域一致、启动成本低）。  
- 可选设置：**「职场加浓」**——仅抽取带职场标签的卡片（仍为六级范围内词条与固定表达，不跨级）。

## 4. 信息架构

### 4.1 推荐结构（三 Tab）

```
┌─────────────────────────────────────┐
│   今日    │   六级库   │    我的     │
└─────────────────────────────────────┘
```

| Tab | 职责 | 支柱 |
|-----|------|------|
| **今日** | 一键短会话（随机序列）；完成「微目标」即算今日达成；分期支持卡片模板切换或混合 | P1、P2、P4 |
| **六级库** | 浏览/搜索 **唯一词表**；**单线进度**：已见 / 已收藏 / 待巩固（可分期）；按职场标签筛选（轻量主题，非多考试） | P1、P2 |
| **我的** | 收藏（分夹：写作夹、会前夹、补丁夹等）、每日张数、提醒、轻量统计 | P3、P4 |

### 4.2 与「极简单核」的折中

首版可 **今日全屏流 + 弱入口** 进入六级库与我的；或 **M0 真·单屏** 验证留存后再加 Tab。原则：**不增加空壳「口语/阅读」Tab**，听读以卡片后链或二级页形式分期接入。

## 5. 功能需求摘要

### 5.1 今日流（P1 + P2 + P4）

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FE-01 | 进入「今日」即开始随机序列，**词源仅为固定六级词域** | P0 |
| FE-02 | 每张卡片至少：拼写、释义（默认中文或产品统一策略）、发音入口；例句/音标有则展示 | P0 |
| FE-03 | 下一张/上一张或滑动；**单次会话默认张数可配置**（如 3～10），划满或主动结束即一次会话 | P0 |
| FE-04 | 翻面或点击展开释义（交互择一） | P1 |
| FE-05 | 会话结束：**一行小结**（今日接触词数等）；**微完成**弹层或文案（与视角 B 一致） | P1 |

### 5.2 收藏（P3）

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FE-10 | 卡片红心：收藏/取消；状态即时可见 | P0 |
| FE-11 | 「我的」中收藏列表；支持 **分夹或标签**（写作/会前/补丁）；去重 | P0 |
| FE-12 | （二期）从收藏生成复习队列 / 小测仅考收藏 | P2 |

### 5.3 六级库（P1 + P2）

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FE-20 | 词表浏览、搜索 | P1 |
| FE-21 | **单线进度**展示（覆盖率、收藏相关指标等，具体指标评审定） | P1 |
| FE-22 | 按职场标签筛选（分期） | P2 |

### 5.4 我的与设置（P3 + P4）

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FE-30 | 收藏入口、基础设置（通知、每次张数、「职场加浓」开关等） | P1 |
| FE-31 | 轻量统计：今日接触词数、累计收藏、六级覆盖率（少而解释性强；详细数据可折叠） | P1 |

### 5.5 非功能期望

- **离线**：词库内置时期望无网可浏览卡片与收藏（与实现一致即可）。  
- **性能**：卡片切换流畅，收藏不阻塞动画。  
- **无障碍**：可点击区域足够大；收藏态色觉友好。

## 6. 里程碑规划

| 阶段 | 交付 |
|------|------|
| **M0** | 固定六级词库 + 随机词条卡流 + 收藏 + 收藏列表 + 微完成定义；可选单屏首发（极简单核） |
| **M1** | 三 Tab 或等价 IA；六级库：搜索 + 单线进度；本地提醒；会话结束小结 |
| **M2** | 间隔复习或加权随机；句块/脚本/补丁等模板按包上线；小测优先绑收藏与低掌握 |
| **M3** | 跟读/TTS；会前模式（只刷收藏 + 计时）；AI 类能力单独立项 |

## 7. 成功指标（建议）

| 类型 | 指标 | 说明 |
|------|------|------|
| 激活 | 首日完成最短会话（如 ≥3 张）比例 | 验证低门槛 |
| 粘性 | 7 日内 ≥3 天有会话 | 早期比 DAU 绝对值更敏感 |
| 价值 | 人均收藏数、收藏 **7 日内二次打开/复习** 比例 | 验证「真用上」 |
| 进度 | **六级单线覆盖率**（对内外一句能说清） | 与「只做一个级别」一致 |
| 体验 | 会话中断率、单会话卡片数分布 | 避免把过长会话当唯一成功 |

**首期不把「词汇量覆盖」作为唯一主 KPI**，与纯备考工具区分。

## 8. 风险与对策

| 风险 | 对策 |
|------|------|
| 用户不考级，排斥「六级」字样 | 对外可用 **「职场英语 · 六级难度锚点」** 等副标题，强调够用与职场 |
| 「背单词」心智与句块/脚本冲突 | 统一叙事 **「学英语」**；句块/脚本标注「能直接用的表达」 |
| 卡片类型增多导致碎 | **M0 仅词条卡**；类型通过内容与标签扩展，不增加主导航项 |
| 与历史「多级别 PRD」用户心智冲突 | 商店与首次启动简短说明；若曾发布多级别版，评估是否需要迁移策略（另评） |
