# WhyMe 英语学习应用 PRD 文档

## 1. 产品概览

WhyMe 是一款专注于英语单词学习的移动应用，通过科学的学习方法帮助用户高效掌握英语单词。应用采用间隔重复算法，结合个性化学习计划，为用户提供定制化的学习体验。

**核心价值：**
- 科学的间隔重复记忆算法
- 个性化学习计划
- 直观的学习进度跟踪
- 丰富的学习模式（学习、复习、测试）

## 2. 核心功能

### 2.1 功能模块

| 模块 | 功能描述 |
|------|----------|
| **学习模块** | 学习新单词，包含发音、释义、例句等内容 |
| **复习模块** | 复习已学单词，巩固记忆 |
| **测试模块** | 检测学习效果，评估掌握程度 |
| **词库管理** | 选择不同难度级别的词库 |
| **个人中心** | 学习统计、收藏管理、设置等 |

### 2.2 页面结构

| 页面 | 模块 | 功能描述 |
|------|------|----------|
| 首页 | 学习概览 | 显示学习进度、今日目标、快速入口 |
| 学习页 | 学习模块 | 学习新单词，包含发音、释义、例句 |
| 复习页 | 复习模块 | 复习已学单词，根据记忆曲线安排 |
| 测试页 | 测试模块 | 多题型测试，评估学习效果 |

## 3. 详细功能需求

### 3.1 学习功能（Learn）

#### 3.1.1 功能描述
学习新单词，通过多维度展示帮助用户理解和记忆单词。

#### 3.1.2 核心流程
1. **词库选择**：用户选择要学习的词库级别
2. **单词展示**：按顺序展示未学习的单词
3. **内容展示**：
   - 单词拼写
   - 发音功能（支持点击播放）
   - 中文释义
   - 英文例句
   - 词性标注
4. **学习操作**：
   - 标记已理解
   - 跳过当前单词
   - 收藏单词
5. **进度保存**：自动记录学习进度，更新单词状态

#### 3.1.3 界面设计
- **布局**：卡片式布局，突出单词和释义
- **交互**：滑动切换单词，点击发音
- **反馈**：学习完成后显示学习成果

#### 3.1.4 技术实现
- **数据来源**：从 `words` 表获取未学习单词
- **进度记录**：更新 `user_word_progress` 表，设置 `isLearned = true`，记录 `learnedAt`
- **算法**：随机选择未学习单词，确保学习的随机性

### 3.2 复习功能（Review）

#### 3.2.1 功能描述
根据间隔重复算法，复习已学但未完全掌握的单词。

#### 3.2.2 核心流程
1. **复习计划**：根据记忆曲线自动生成复习计划
2. **单词展示**：按复习优先级展示单词
3. **复习模式**：
   - 看单词回忆释义
   - 看释义回忆单词
   - 听力练习
4. **难度评估**：
   - 用户评估记忆难度（容易、一般、困难）
   - 系统根据评估调整下次复习时间
5. **进度更新**：更新复习次数、掌握级别、下次复习时间

#### 3.2.3 界面设计
- **布局**：问答式布局，突出问题和选项
- **交互**：点击选择答案，滑动切换单词
- **反馈**：显示正确答案，提供发音

#### 3.2.4 技术实现
- **数据来源**：从 `user_word_progress` 表获取需要复习的单词（`nextReviewDate <= today`）
- **算法**：间隔重复算法，根据掌握级别调整复习间隔
- **进度记录**：更新 `masteryLevel`、`reviewCount`、`nextReviewDate` 等字段

### 3.3 测试功能（Quiz）

#### 3.3.1 功能描述
通过多种题型测试用户对单词的掌握程度。

#### 3.3.2 核心流程
1. **测试设置**：选择测试范围（最近学习、特定级别等）
2. **题型选择**：
   - 选择题（多选一）
   - 填空题
   - 听力题
   - 拼写题
3. **测试过程**：
   - 限时答题
   - 实时反馈
   - 错题标记
4. **测试结果**：
   - 正确率统计
   - 错题分析
   - 薄弱环节提示

#### 3.3.3 界面设计
- **布局**：考试式布局，清晰展示题目和选项
- **交互**：点击选择答案，提交后显示结果
- **反馈**：测试结束后显示详细报告

#### 3.3.4 技术实现
- **数据来源**：从 `user_word_progress` 表获取已学习单词
- **评分系统**：根据答题情况计算得分和正确率
- **结果分析**：统计错误类型，生成个性化建议

## 4. 用户流程

### 4.1 学习流程
1. 用户打开应用 → 登录 → 进入首页
2. 点击「开始学习」按钮 → 选择词库级别
3. 进入学习页面 → 浏览单词内容
4. 点击发音 → 听单词发音
5. 理解后点击「已理解」→ 进入下一个单词
6. 完成学习计划 → 显示学习成果

### 4.2 复习流程
1. 用户打开应用 → 进入首页
2. 点击「复习」按钮 → 进入复习页面
3. 查看单词 → 回忆释义
4. 选择记忆难度 → 系统调整复习计划
5. 完成复习 → 显示复习结果

### 4.3 测试流程
1. 用户打开应用 → 进入学习页面
2. 点击「测试」选项 → 选择测试范围
3. 进入测试页面 → 回答问题
4. 完成测试 → 查看测试报告
5. 查看错题 → 针对性复习

## 6. 技术实现要点

### 6.1 数据结构
- **`words` 表**：存储单词基础信息（id, word, phonetic, definition, example, level）
- **`user_word_progress` 表**：存储用户学习进度（wordId, masteryLevel, isLearned, nextReviewDate, reviewCount, correctCount, lastReviewResult, createdAt, learnedAt, lastReviewedAt）

### 6.2 核心算法
- **间隔重复算法**：根据记忆曲线计算下次复习时间
- **学习计划生成**：根据用户目标和可用时间生成每日学习计划
- **测试题目生成**：根据用户掌握情况生成个性化测试题目

### 6.3 技术栈
- **前端**：Jetpack Compose
- **后端**：本地数据库（Room）
- **状态管理**：ViewModel + Flow
- **依赖注入**：Hilt

## 7. 性能优化

### 7.1 数据库优化
- 使用索引加速查询（wordId, nextReviewDate, masteryLevel, isLearned）
- 批量操作减少数据库访问次数
- 合理使用事务确保数据一致性

### 7.2 界面优化
- 懒加载列表减少内存使用
- 图片和音频资源缓存
- 异步加载数据避免 UI 阻塞
