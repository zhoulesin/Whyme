# WhyMe 项目结构索引

## 1. 文档目的

这份文档不是详细设计说明，而是后续改代码时的“快速定位地图”。

适用场景：
- 想先知道项目按什么层次组织
- 想快速找到某个功能对应的文件
- 想判断一个改动应该落在 UI、domain 还是 data

当前仓库实际形态：
- 单模块 Android 项目
- 技术栈：Kotlin + Jetpack Compose + Hilt + Room + DataStore
- Gradle 工程名：`Whyme`
- 唯一业务模块：`app`

## 4. 核心代码分层

项目基本是标准三层：

### 4.1 入口层

- `app/src/main/java/com/zhoulesin/whyme/WhyMeApplication.kt`
  Hilt 应用入口。
- `app/src/main/java/com/zhoulesin/whyme/MainActivity.kt`
  App Compose 入口，负责：
  - 初始化 `Scaffold`
  - 管理底部导航显示
  - 挂载 `AppNavHost`

### 4.2 `ui/` 表现层

负责：
- Compose 页面
- ViewModel
- 导航
- 通用 UI 组件
- 主题样式

### 4.3 `domain/` 领域层

负责：
- 领域模型定义
- Repository 接口
- UseCase 业务用例
- 学习状态、复习算法等核心规则

### 4.4 `data/` 数据层

负责：
- Room 数据库
- DAO
- Entity / Mapper
- DataStore 偏好存储
- Repository 实现
- 启动时词库初始化

### 4.5 `di/` 注入层

负责：
- 数据库对象提供
- Repository 绑定

## 6. 各目录职责说明

### 6.1 `ui/home`

首页相关代码。

- `HomeScreen.kt`
  首页 UI，展示欢迎语、今日进度、统计卡片、待复习提醒。
- `HomeViewModel.kt`
  首页聚合数据来源，组合：
  - 用户统计
  - 每日目标
  - 今日记录
  - 当前词库级别的待复习词

### 6.2 `ui/learning`

学习主流程核心区域。

- `LearningScreen.kt`
  学习中心入口页，选择新词学习 / 复习 / 测试。
- `LearningSessionScreen.kt`
  真正执行学习会话的页面，负责根据 mode 启动流程。
- `LearningViewModel.kt`
  学习流程核心状态管理：
  - 加载今日新词/待复习词
  - 开始学习、复习、测试
  - 单词答题/标记结果
  - 保存学习进度
  - 更新学习记录
- `QuizScreen.kt`
  独立测试页。
- `WordDetailScreen.kt`
  单词详情页。

### 6.3 `ui/profile`

个人中心页面与状态。

- `ProfileScreen.kt`
- `ProfileViewModel.kt`

### 6.4 `ui/favorites`

收藏列表。

- `FavoritesScreen.kt`
- `FavoritesViewModel.kt`

### 6.5 `ui/statistics`

学习数据统计页。

- `StatisticsScreen.kt`
- `StatisticsViewModel.kt`

### 6.6 `ui/wordbank`

词库级别切换相关状态。

- `WordBankViewModel.kt`
  管理当前学习级别、已启用级别、级别进度。

### 6.7 `ui/navigation`

导航定义。

- `Screen.kt`
  所有路由常量、底部 Tab、学习模式枚举。
- `AppNavHost.kt`
  路由注册与页面跳转。

### 6.8 `ui/components`

可复用组件。

- `ProgressComponents.kt`
  进度环、统计项等。
- `WordCard.kt`
  单词卡片、翻面展示、掌握度按钮。
- `WordLevelSelector.kt`
  词库级别切换组件。

### 6.9 `domain/model`

核心业务模型。

- `Word.kt`
  单词领域模型。
- `LearningRecord.kt`
  学习记录、用户统计、每日目标。
- `LearningState.kt`
  学习状态机、测试状态、复习结果。
- `MasteryLevel.kt`
  掌握等级、复习间隔、每日限制。
- `WordLevel.kt`
  词库分级、用户词库设置、级别进度。
- `Achievement.kt`
  成就定义。

### 6.10 `domain/usecase`

业务动作入口。

- `WordUseCases.kt`
  查新词、查复习词、收藏、搜索、更新复习结果。
- `LearningUseCases.kt`
  今日记录、用户统计、每日目标、记录学习会话。
- `ReviewIntervalCalculator.kt`
  复习间隔与掌握等级变化规则。

### 6.11 `data/local`

本地存储相关。

- `AppDatabase.kt`
  Room 数据库定义。
- `AppInitializer.kt`
  应用启动时检查是否需要初始化词库。
- `DatabaseInitializer.kt`
  从 `assets/words/*.json` 导入词库。
- `PreferencesDataStore.kt`
  存储每日目标、连续打卡、学习进度、初始化标记等。

### 6.12 `data/local/dao`

数据库访问入口。

- `WordDao.kt`
  单词相关查询、搜索、收藏、复习更新、统计。
- `LearningRecordDao.kt`
  学习记录增删改查与汇总。
- `UserWordBankSettingsDao.kt`
  词库设置与级别进度。

### 6.13 `data/local/entity`

数据库实体和映射。

- `WordEntity.kt`
  `words` 表结构。
- `LearningRecordEntity.kt`
  `learning_records` 表结构。
- `UserWordBankSettingsEntity.kt`
  当前词库配置表。
- `LevelProgressEntity.kt`
  词库级别进度表。
- `WordMapper.kt`
  `WordEntity <-> Word`
- `LearningRecordMapper.kt`
  `LearningRecordEntity <-> LearningRecord`
- `WordJson.kt`
  assets 中词库 JSON 对应的数据结构。

### 6.14 `data/repository`

领域接口的本地实现。

- `WordRepositoryImpl.kt`
  对接 `WordDao`
- `LearningRepositoryImpl.kt`
  对接 `LearningRecordDao + PreferencesDataStore`
- `WordBankRepositoryImpl.kt`
  对接级别设置和进度 DAO

### 6.15 `di`

依赖注入模块。

- `DatabaseModule.kt`
  提供 `AppDatabase` 和各 DAO。
- `RepositoryModule.kt`
  绑定 Repository 接口到实现类。

## 10. 当前阅读后的几个判断

### 10.1 这是一个“单模块 + 分层清晰”的项目

优点是定位成本低。绝大多数改动都能沿着下面这条链快速找到：

`Screen -> ViewModel -> UseCase -> Repository -> DAO`

### 10.2 学习主流程的核心文件非常集中

后续如果要改业务行为，优先看这几个文件：

- `ui/learning/LearningViewModel.kt`
- `ui/learning/LearningSessionScreen.kt`
- `domain/usecase/ReviewIntervalCalculator.kt`
- `data/repository/WordRepositoryImpl.kt`
- `data/repository/LearningRepositoryImpl.kt`

### 10.3 词库和统计是两个独立关注点

- 词库切换：`WordBank*`
- 学习统计：`LearningRecord*` + `Statistics*`

后面如果需求涉及“词库级别”和“学习统计”同时变化，通常需要同时检查两条链路。
